1. FUNCIONAMENTO DO PROGRAMA
================================================================================

1.1 VISÃO GERAL
---------------
O Trampix é uma plataforma web que conecta freelancers e empresas, permitindo:
- Cadastro de usuários com múltiplos perfis (empresa/freelancer)
- Publicação e gerenciamento de vagas de trabalho
- Sistema de candidaturas e seleção
- Perfis públicos e portfolios
- Autenticação segura e autorização baseada em roles

1.2 ARQUITETURA TÉCNICA
-----------------------
- Framework: Laravel 12.x (PHP 8.2+)
- Padrão: MVC (Model-View-Controller)
- Autenticação: Laravel Breeze
- Frontend: Blade Templates + TailwindCSS + Alpine.js
- Banco de Dados: SQLite (desenvolvimento)
- Build Tool: Vite
- Testes: PHPUnit

1.3 FLUXO DE EXECUÇÃO PRINCIPAL
-------------------------------

A) AUTENTICAÇÃO E REGISTRO:
   1. Usuário acessa /login ou /register
   2. Laravel Breeze processa autenticação
   3. Middleware 'auth' protege rotas privadas
   4. Sistema define role (empresa/freelancer/admin)

B) GESTÃO DE PERFIS:
   1. Usuário autenticado acessa /profile
   2. ProfileController gerencia edição unificada
   3. Policies verificam permissões (CompanyPolicy/FreelancerPolicy)
   4. Dados salvos via Eloquent ORM

C) SISTEMA DE VAGAS:
   1. Empresa cria vaga via JobVacancyController
   2. Middleware 'can:isCompany' autoriza ação
   3. Vaga exibida publicamente em /vagas
   4. Freelancers podem candidatar-se

D) PROCESSO DE CANDIDATURA:
   1. Freelancer acessa vaga específica
   2. ApplicationController processa candidatura
   3. Empresa visualiza candidatos
   4. Status da candidatura é atualizado

1.4 TECNOLOGIAS INTEGRADAS
---------------------------
- Laravel Eloquent: ORM para relacionamentos complexos
- Laravel Policies: Autorização granular
- TailwindCSS: Framework CSS utilitário
- Alpine.js: Reatividade frontend leve
- Vite: Build tool moderno e rápido
- SQLite: Banco de dados embarcado

5. ARQUITETURA DE DADOS
================================================================================

5.1 RELACIONAMENTOS PRINCIPAIS
------------------------------
User (1:N) → Company → (1:N) JobVacancy → (1:N) Application
User (1:N) → Freelancer → (N:N) Skill
Freelancer (1:N) → Application ← (N:1) JobVacancy

5.2 TABELAS E CAMPOS PRINCIPAIS
-------------------------------
users: id, name, email, password, role, email_verified_at
companies: id, user_id, name, cnpj, sector, location, is_active
freelancers: id, user_id, bio, portfolio_url, cv_url, is_active
job_vacancies: id, company_id, title, description, status
applications: id, freelancer_id, job_vacancy_id, status
skills: id, name
freelancer_skill: freelancer_id, skill_id (pivot)
